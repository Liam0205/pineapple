# 逻辑表 (Logical Table) 示例

## 场景

一个推荐服务中，召回团队和排序团队共同处理同一批 items。
召回团队负责从 1000 个候选 item 中筛选出 200 个，排序团队只关心这 200 个。

## 物理表 (全量数据)

```
Item 物理表 (1000 行):
┌──────────┬────────┬────────┬──────────┬──────────┐
│ item_id  │ ctr    │ cvr    │ score    │ rank     │
├──────────┼────────┼────────┼──────────┼──────────┤
│ 1001     │ 0.12   │ 0.05   │ nil      │ nil      │
│ 1002     │ 0.08   │ 0.03   │ nil      │ nil      │
│ 1003     │ 0.15   │ 0.07   │ nil      │ nil      │
│ ...      │ ...    │ ...    │ ...      │ ...      │  ← 1000 行
└──────────┴────────┴────────┴──────────┴──────────┘
```

## 召回团队的逻辑表

召回算子处理完后，从 1000 个 item 中筛选出 200 个。
排序团队不需要看到被淘汰的 800 个。

```python
# 召回阶段 DSL
flow.recall_op(...)  # 产出 1000 -> 200

# 创建逻辑表: 只包含召回通过的 200 个 item
# 后续排序算子基于这个逻辑表操作
flow.create_view(
    name="recalled_items",
    filter="recall_passed == true",
)
```

```
逻辑表 "recalled_items" (200 行, 底层指向物理表的子集):
┌──────────┬────────┬────────┬──────────┬──────────┐
│ item_id  │ ctr    │ cvr    │ score    │ rank     │
├──────────┼────────┼────────┼──────────┼──────────┤
│ 1001     │ 0.12   │ 0.05   │ nil      │ nil      │  ← 物理表第 0 行
│ 1003     │ 0.15   │ 0.07   │ nil      │ nil      │  ← 物理表第 2 行
│ ...      │ ...    │ ...    │ ...      │ ...      │  ← 共 200 行
└──────────┴────────┴────────┴──────────┴──────────┘
```

## 排序团队的逻辑表

排序算子在 "recalled_items" 上操作，写入 score 和 rank。
**写入会穿透到物理表**（与数据库只读视图不同）。

```python
# 排序阶段 DSL, 操作的是逻辑表
flow.rank_op(
    view="recalled_items",
    item_input=["ctr", "cvr"],
    item_output=["score", "rank"],
)
```

```
排序后, 逻辑表和物理表同时更新:

逻辑表 "recalled_items":
┌──────────┬────────┬────────┬──────────┬──────────┐
│ item_id  │ ctr    │ cvr    │ score    │ rank     │
├──────────┼────────┼────────┼──────────┼──────────┤
│ 1001     │ 0.12   │ 0.05   │ 0.95     │ 2        │
│ 1003     │ 0.15   │ 0.07   │ 0.98     │ 1        │
│ ...      │ ...    │ ...    │ ...      │ ...      │
└──────────┴────────┴────────┴──────────┴──────────┘

物理表 (对应行也被更新):
┌──────────┬────────┬────────┬──────────┬──────────┐
│ 1001     │ 0.12   │ 0.05   │ 0.95     │ 2        │  ← 已更新
│ 1002     │ 0.08   │ 0.03   │ nil      │ nil      │  ← 未被召回, 无变化
│ 1003     │ 0.15   │ 0.07   │ 0.98     │ 1        │  ← 已更新
│ ...      │ ...    │ ...    │ ...      │ ...      │
└──────────┴────────┴────────┴──────────┴──────────┘
```

## 核心特点

1. **逻辑表是物理表的"窗口"**, 不复制数据, 只记录行索引的映射。
2. **可读写**: 对逻辑表的写入穿透到物理表 (区别于数据库只读视图)。
3. **划分操作空间**: 不同团队/阶段只看到自己关心的数据子集, 互不干扰。
4. **可嵌套**: 可以基于逻辑表再创建逻辑表 (如排序后再做一次截断)。
