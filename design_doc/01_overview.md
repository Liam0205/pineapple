# Pineapple 计算引擎 — 概述

## 定位

面向 **搜索/推荐/广告** 业务的通用计算引擎。

## 目标用户

| 角色 | 使用方式 |
|------|----------|
| 工程架构同学 | 开发新的算子 (Operator)，增强引擎能力 |
| 算法同学 | 用 Python DSL 编写业务逻辑 |

## 技术栈

- **引擎核心**: Go
- **DSL**: Python
- **配置格式**: JSON（由 Python DSL 生成）
- **嵌入脚本**: Lua（增强通用算子的表达能力）

## 运行流程

```
Python DSL  ──(执行)──▶  JSON 配置文件
                              │
                              ▼
               Go 引擎解析 JSON 配置
                              │
                              ▼
              解析算子依赖关系，构建 DAG
                              │
                              ▼
                  基于 DAG 并行执行算子
```

## 核心设计要点

1. **算子 (Operator)** 是基本计算单元，由 Go 实现。
2. **Python DSL** 是面向算法同学的声明式接口；运行 DSL 产出 JSON 配置，不参与运行时计算。
3. **JSON 配置** 是引擎与 DSL 之间的契约；引擎据此解析算子依赖，构建 DAG。
4. **DAG 调度** 引擎分析算子间的依赖关系，形成有向无环图，尽可能并行执行无依赖的算子。
5. **Lua 嵌入** 通用算子可内嵌 Lua 运行时，在不新增 Go 算子的情况下实现特定逻辑。
