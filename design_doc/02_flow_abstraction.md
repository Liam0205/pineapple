# 流程抽象

## 算子 (Operator)

算子是 Pineapple 的基本计算单元。

### 分类

| 类型 | 维护方 | 说明 |
|------|--------|------|
| 通用算子 | 工程架构团队 | 过滤、召回、模型预估等，各业务直接复用 |
| 自定义算子 | 业务/算法团队 | 满足通用算子无法覆盖的定制化逻辑 |
| Lua 增强算子 | 业务/算法团队 | 通用算子内嵌 Lua 脚本，无需编写新 Go 算子即可实现特定逻辑 |

### 算子接口（初步）

每个算子需要声明：
- **输入数据**: 依赖哪些数据字段
- **输出数据**: 产出哪些数据字段
- **配置参数**: 算子行为的可配置项

## DAG 构建

采用 **数据驱动的隐式构图**：

1. 算子在配置中声明自己的输入和输出数据字段。
2. 引擎根据各算子的输入/输出依赖关系，自动推导出 DAG 拓扑。
3. 无数据依赖的算子自动并行执行。

### 示例

```
算子 A: 输出 [feature_x]
算子 B: 输入 [feature_x], 输出 [score_b]     ─┐
算子 C: 输入 [feature_x], 输出 [score_c]      ├─ B、C 并行
算子 D: 输入 [score_b, score_c], 输出 [rank]  ─┘ D 等待 B、C 完成
```

推导出的 DAG：
```
A ──▶ B ──▶ D
 └──▶ C ──┘
```

### 全流程漂移

由于 DAG 完全由数据依赖决定，算子在 DSL 中的编排顺序改变时，DAG 结构会自动重新推导。这使得流程编排非常灵活。

## DAG 调度

- 基于拓扑排序执行。
- 无依赖的算子并行调度。
- 目标：无锁设计，支持高并发场景。

## 分层解耦

```
┌─────────────────────────────┐
│   算法工作空间 (Python DSL)   │  编排算子、提交配置
├─────────────────────────────┤
│        JSON 配置 (契约)       │
├─────────────────────────────┤
│   架构工作空间 (Go 算子)      │  实现算子、提供引擎二进制
└─────────────────────────────┘
```

算法团队与架构团队通过 JSON 配置解耦，互不干扰。
